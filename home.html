Critical Rendering Path - Ruta Critica de Procesamiento

Critical Render Path se refiere a la secuencia de pasos que el navegador realiza para recibir código en HTML, CSS y JavaScript y convertirlos en la vista inicial de una página web.

La ruta de procesamiento crítica es la secuencia de pasos que sigue el navegador para convertir HTML, CSS y JavaScript en píxeles en la pantalla.

La ruta de renderización crítica incluye el modelo de objetos de documento ( el modelo de objetos CSS ) el árbol de renderización y el diseño

DOM -Document Object Model

La construcción de DOM es incremental La respuesta HTML se convierte en tokens que se convierten en NODOS que se convierten en el árbol DOM Un solo nodo DOM comienza con un token startTag y termina con un token endTag Los nodos contienen toda la información relevante sobre el elemento HTML La información se describe mediante tokens Los nodos están conectados en un árbol DOM según la jerarquía de tokens Si otro conjunto de tokens startTag y endTag se encuentra entre un conjunto de startTag y endTags tiene un nodo dentro de un nodo, que es como definimos la jerarquía del árbol DOM.

Cuanto mayor sea el número de nodos, más tardarán los siguientes eventos en la ruta de renderización crítica ..¡La medida! Unos pocos nodos adicionales no marcarán la diferencia, pero la divitis puede provocar jank

- Interpreta el HTML.
- Estructura los nodos en forma de arbol.
- El modelo se Puede Modificar.

CSSOM - CSS Object Model

El CSSOM contiene todos los estilos de la página información sobre cómo diseñar ese DOM. CSSOM es similar al DOM, pero diferente Si bien la construcción del DOM es incremental, CSSOM no lo es

CSS el navegador bloquea el procesamiento de la página hasta que recibe y procesa todo el CSS. CSS bloquea el procesamiento porque las reglas se pueden sobrescribir, por lo que el contenido no se puede procesar hasta que el CSSOM esté completo

WEB API - Application Programming Interface

Un Application Programming Interface (Interfaz de programación de aplicaciones), abreviado como API es un conjunto de subrutinas, funciones y procedimientos (o métodos, en la programación orientada a objetos) que ofrece cierta biblioteca para ser utilizado por otro software como una capa de abstracción

Uno de los principales propósitos de una API consiste en proporcionar un conjunto de funciones de uso general por ejemplo, para dibujar ventanas o iconos en la pantalla De esta forma, los programadores se benefician de las ventajas del API haciendo uso de su funcionalidad, evitándose el trabajo de programar todo desde el principio
